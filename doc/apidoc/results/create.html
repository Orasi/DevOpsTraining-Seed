POST /results
Create new result

Creates a result for project based on API KEY

Metadata

--- Unauthenticated path for automated results,  User-Token header is not required.  For
  manual result User-Token header is required

Params

Param name Description
User-Token
required

Authentication token. Must be provided in the header of all calls unless otherwise indicated

Validations:

  • Must be String
result
required

Validations:

  • Must be a Hash
result[status]
required

Result status

Validations:

  • Must be one of: pass, fail, skip.
result[environment_id]
required

Environment UUID

Validations:

  • Must be String
result[testcase_id]
required

Either the testcase validation id or the testcase name

Validations:

  • Must be one of: String, number.
result[result_type]
required

Result type

Validations:

  • Must be one of: automated, manual.
result[project_id]
optional , nil allowed

Project API Key. Required for automated result type

Validations:

  • Must be String
result[execution_id]
optional , nil allowed

Execution ID. Required for Manual result type

Validations:

  • Must be a number.
result[comment]
optional , nil allowed

Comment

Validations:

  • Must be String
result[stacktrace]
optional , nil allowed

Stacktrace

Validations:

  • Must be String
result[link]
optional , nil allowed

Link to external result

Validations:

  • Must be String
result[screenshot]
optional , nil allowed

Base64 encoded screenshot

Validations:

  • Must be String
result[step_log]
optional , nil allowed

JSON Step Log of test

Validations:

  • Must be String
