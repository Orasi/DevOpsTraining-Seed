GET /users/:id
User details

Returns details of a single user. Admins can see any users. Non-admins can only view themselves

Supported Formats

json

Examples

GET /users/5680
200
{
  "user": {
    "id": 5680,
    "username": "edward_beahan",
    "email": "bettie.krajcik@wuckert.info",
    "first_name": "Kyleigh",
    "last_name": "Davis",
    "company": "Spencer, Schamberger and Ratke",
    "admin": null,
    "created_at": "2017-05-17T16:26:37.894Z",
    "updated_at": "2017-05-17T16:26:37.894Z"
  }
}

Params

Param name Description
User-Token
required

Authentication token. Must be provided in the header of all calls unless otherwise indicated

Validations:

  • Must be String
id
required

User ID

Validations:

  • Must be a number.
