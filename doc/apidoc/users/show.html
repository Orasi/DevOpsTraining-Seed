GET /users/:id
User details

Returns details of a single user. Admins can see any users. Non-admins can only view themselves

Supported Formats

json

Examples

GET /users/30736
200
{
  "user": {
    "id": 30736,
    "username": "imelda",
    "email": "claudia@corwin.io",
    "first_name": "Caitlyn",
    "last_name": "Schmitt",
    "company": "Smitham, McKenzie and Effertz",
    "admin": null,
    "created_at": "2017-03-03T14:34:12.963Z",
    "updated_at": "2017-03-03T14:34:12.963Z"
  }
}

Params

Param name Description
User-Token
required

Authentication token. Must be provided in the header of all calls unless otherwise indicated

Validations:

  • Must be String
id
required

User ID

Validations:

  • Must be a number.
