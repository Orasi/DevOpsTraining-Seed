PUT /users/:id
Update existing user

Only accessible by admin users

Examples

PUT /users/30742
{
  "user": {
    "first_name": "Tom",
    "last_name": "Jones",
    "username": "tom.jones@singing.com",
    "company": "singing",
    "admin": true
  }
}
200
{
  "user": {
    "id": 30742,
    "username": "tom.jones@singing.com",
    "email": "luz@wilderman.io",
    "first_name": "Tom",
    "last_name": "Jones",
    "company": "singing",
    "admin": true,
    "created_at": "2017-03-03T14:34:13.060Z",
    "updated_at": "2017-03-03T14:34:13.077Z"
  }
}

Param name Description
User-Token

Authentication token. Must be provided in the header of all calls unless otherwise indicated

  • Must be String

required

User ID

Validations:

  • Must be a number.

Validations:

  • Must be a Hash

User's First name

Validations:

Validations:

  • Must be String

User's Company

Validations:

  • Must be String

Validations:

  • Must be String

Validations:
