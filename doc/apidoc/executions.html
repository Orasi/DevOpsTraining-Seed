POST /executions/close
Close execution

Closes execution and opens a new execution

Metadata

--- |-
  User must provide either execution_id or project_key
          If project_key is provided the open execution will be closed
          Only accessible if project is viewable by current user
          Displays details of NEW execution

Examples

POST /executions/close
{
  "project_key": "26673b4c11e984a216c532fd3afe7146",
  "name": "Some Name"
}
200
{
  "execution": {
    "id": 12407,
    "created_at": "2017-03-03T14:34:04.295Z",
    "updated_at": "2017-03-03T14:34:04.295Z",
    "project_id": 12298,
    "closed": false,
    "closed_at": null,
    "deleted": null,
    "name": "Some Name"
  }
}

Params

Param name Description
User-Token
required

Authentication token. Must be provided in the header of all calls unless otherwise indicated

Validations:

  • Must be String
project_key
optional

Project Key. Required if execution_id is not present

Validations:

  • Must be String
execution_id
optional

Execution ID. Required if project_key is not present

Validations:

  • Must be a number.
