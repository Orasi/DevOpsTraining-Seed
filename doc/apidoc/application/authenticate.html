POST /authenticate
Login

Returns User-Token and current user details. Allows user to login with either username or email address

Metadata

--- Unauthenticated path.  User-Token header is not required
...

Examples

POST /authenticate/
{
  "username": "joan.wiza",
  "password": 12345
}
200
{
  "user": {
    "id": 30642,
    "username": "joan.wiza",
    "email": "ansley@dicki.name",
    "first_name": "Alberta",
    "last_name": "Quitzon",
    "token": "vnYYFkK8F31oFHzb9MhwvFjs",
    "admin": null
  }
}

Params

username
required

User username

Validations:

Validations:
